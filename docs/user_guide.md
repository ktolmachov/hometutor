# hometutor — руководство пользователя

Актуализировано: 2026-07-18.

> Положите учебные материалы в `data/`, проиндексируйте их и получите локального учебного помощника: ответы с источниками, тьютора, quiz, flashcards, SM-2 повторения, прогресс, Smart Study Router, AI Agent Coach и «Живой конспект» — сборку рабочих конспектов из точных разделов лекций.

## Для кого это

`hometutor` полезен, если у вас есть лекции, конспекты, статьи, Obsidian vault или учебные PDF, и вы хотите не просто “поговорить с документом”, а пройти цикл обучения.

```text
data/ с материалами
  -> индекс
  -> ответ с источниками
  -> tutor-разбор
  -> quiz
  -> flashcards + SM-2
  -> progress + Smart Study Router
  -> Knowledge Graph -> точные разделы -> Живой конспект
  -> AI Agent Coach -> учебная сессия / пробелы графа / коуч конспекта
```

Проект local-first: материалы, индекс и учебное состояние находятся на вашей машине. По умолчанию `config.env` направляет LLM и embeddings на loopback endpoints (`127.0.0.1`). Если вы явно переопределяете `EMBED_API_BASE`, `LLM_API_BASE` или fallback-настройки на облачный provider, содержимое запросов может уходить внешнему провайдеру (см. [quickstart.md](quickstart.md)).

## Первый запуск (онбординг)

При первом входе `hometutor` проверяет три вещи до вопроса: доступен ли API, есть ли материалы в индексе и отвечает ли локальная модель. Если API недоступен, Mission Control показывает действие для запуска сервера. Если модель выключена, вы увидите подсказку запустить LM Studio или совместимый OpenAI-compatible сервер и загрузить модель из настроек.

Статус «готовится / собирается» (для первого сеанса) показывается **только** при реально идущем реиндексе и включённом `enable_first_session_precompute` (по умолчанию true). При выключенном флаге или без активного реиндекса — нейтральная подсказка без ложных обещаний.

Если индекс пуст, главная показывает три входа:

- **Загрузить файлы** — выберите `.md`, `.txt`, `.pdf`, `.docx` или `.html`; UI сохранит их в `data/uploads/` и запустит переиндексацию.
- **Попробовать демо-материалы** — учебные файлы из `demo_data/` копируются в
  `data/demo/`, а встроенный курс `hometutor_101` — в
  `data/uploads/hometutor_101/` вместе с `*.media.json` и MP4 для панели
  «🎞 Все видео урока». После индексации появятся готовые стартовые вопросы.
- **У меня уже есть папка** — положите материалы в показанный путь `data/` и нажмите «Переиндексировать».

Демо-режим помечается баннером «демо-песочница». Его можно удалить одной кнопкой: если после удаления не осталось материалов, активный индекс очищается до пустого состояния, чтобы демо-документы больше не участвовали в ответах.

## Главная: Mission Control

Первый экран — не landing page, а рабочая панель. Над hero идёт компактная строка контекста: активный курс и XP/стрик. Дальше она показывает:

- Smart Study Router: один рекомендуемый следующий шаг (hero) и объяснение “почему сейчас” — тьютор, quiz, due-карточки и активный курс подхватываются этой рекомендацией, а не отдельными карточками;
- resume-карточку графа знаний (мини-карта концептов, лекций, доступных узлов и mastery);
- resume-карточку «Живой конспект», если сборка рабочего конспекта не закончена (разделы/лекции/концепты и кнопка «Продолжить сборку»);
- полную сетку режимов (тьютор, quiz, flashcards, темы, курс, адаптивный план, **Агент** при `AGENT_ENABLED=true`), свёрнутую под «Ещё режимы», — чтобы hero оставался чистым, но все входы оставались доступны;
- доступ к истории, метрикам и explain-file.

В строке контекста также может появиться «🗺 Карта отстаёт: N материалов не на карте» — когда индекс уже увидел новые материалы, а граф ещё не перестроен/опубликован (гейт или LLM недоступны). Сегмент убирается автоматически после успешного promote. Это объясняет, почему ответы уже видят свежий материал, а карта/план/конспект — пока нет.

Плитка «Агент» (gated feature) ведёт на dedicated view с префиллом темы из текущего контекста. После прогона доступны кнопки сохранения карточек-кандидатов (по явному действию пользователя) и (в Прогрессе) история запусков.

Главная нужна для возврата: пользователь не вспоминает, где остановился, а продолжает с локального состояния.

## Панель управления и уровни интерфейса

Интерфейс может показывать разный объём возможностей. В сайдбаре и на Mission Control есть кнопка
«Настроить интерфейс»: она открывает панель с пресетами.

| Уровень | Что показывает |
|---|---|
| Учёба (по умолчанию) | Mission Control, быстрый ответ, поиск материалов, объяснение файла, тьютор, quiz, flashcards, прогресс, темы |
| Полный | + курс, адаптивный план, Knowledge Graph, живой конспект, история |
| Диагностика | + метрики, чистый вид, аналитика, backup/QR-перенос, экспертные Q&A-фильтры, debug/trace |

Плитка «Курс» на Mission Control и вкладка «Темы» — точки активации курса — видны на любом уровне,
включая «Учёба»: без этого новый пользователь не мог бы выбрать курс вообще. Активация курса сама
поднимает уровень до «Полный», если сейчас выбран «Учёба», — не нужно отдельно искать панель управления,
чтобы увидеть Course Cockpit, план и граф знаний после выбора курса.

В панели управления есть вкладка **«Оформление»** (третья, между «Интерфейс» и «RAG и ingest»):
5 цветовых схем (миров): 🌲 Лес (по умолчанию), 🌊 Океан, 🌅 Закат, 🚀 Космос, 🍇 Ягода.
Каждый мир меняет акценты, фон приложения, карточки и заголовки. Выбор сохраняется в профиле
и входит в backup. Экран логина не затрагивается — тема применяется после входа.

Существующие пользователи автоматически получают режим «Диагностика», чтобы привычный интерфейс не менялся.
Новому пользователю можно выбрать режим в онбординге: «Учёба», «Полный (курсы и план)» или
«Диагностика (метрики и trace)». Старые сохранённые значения уровня (`"1"`..`"5"`, `"all"`) читаются
прозрачно и мигрируют на новую схему при первом обращении.

На HF Space / локальном демо-режиме (`HOME_RAG_DATA_MODE=demo`) встроенный курс `hometutor_101`
активируется автоматически при первом запуске — если у пользователя ещё нет собственного выбора
курса. Срабатывает один раз; явную деактивацию курса это не переопределяет.

Скрытый раздел не становится недоступным: deep-link, переходы с карточек и «Ещё разделы» всё равно открывают нужную
страницу. В панели есть «Тонкая настройка» — отдельные тумблеры поверх выбранного пресета. Эти настройки хранятся
локально в профиле (`app_kv`) и входят в backup/restore.

Вкладка **RAG и ingest** в той же панели управляет основными и расширенными параметрами retrieval, chunking,
embeddings и ingest. Эффективные значения = `config.env` / `.env` + локальные overrides профиля. Изменения
chunking/embeddings и ingest вступают после переиндексации; параметры retrieval — с следующего запроса.

## Быстрый ответ

Режим для вопроса “что говорится в моих материалах?”.

Вы получаете:

- ответ по индексу;
- список источников;
- кнопку «➕ В Живой конспект» на источниках ответа: если фрагмент уверенно
  сопоставлен с разделом подготовленного markdown-конспекта, он добавляется сразу;
  при низкой уверенности или неоднозначных заголовках UI просит выбрать раздел вручную;
- видео-цитату «🎬 Смотреть с M:SS» на источнике ответа, если тот же guarded resolver
  нашёл раздел, а его `media_sidecar` актуален и даёт доверенный таймкод;
- confidence/trace сигналы;
- мост в tutor-режим;
- debug-информацию при включённых диагностических панелях.

HTTP-контракт: `POST /ask`.

## Чат с тьютором

Tutor работает через тот же endpoint `/ask`, но с `query_mode="tutor"` и learner context. Он может:

- объяснить тему проще;
- дать пример;
- встроить micro-quiz;
- продолжить multi-turn сессию по `session_id`;
- учитывать learner goal snapshot.

Важный принцип: переход из ответа в обучение не должен терять тему, источники и цель.

## AI Agent Coach

Agent Coach — экспериментальный read-only режим поверх того же endpoint `/ask`.
Он включается флагом `AGENT_ENABLED=true` и вызывается через
`query_mode="agent"`. При включённом флаге на Mission Control появляется
плитка «Агент», ведущая на страницу с текстовым вводом вопроса.

Агент пишет только компактную диагностическую историю прогона (`run_id`,
scenario, stop reason, tool calls) в локальный `user_state.db`. Он не сохраняет
карточки, не записывает quiz-result, не меняет граф и не редактирует Живой
конспект. Все карточки, вопросы, правки конспекта и графа — только
draft-предложения для ручного действия пользователя.

Доступные сценарии:

| Сценарий | Когда срабатывает | Что делает |
|---|---|---|
| `study_session` | обычный `query_mode="agent"` вопрос по теме | собирает короткую учебную сессию: диагностика, фокус, план на 10–20 минут, проверочные вопросы, draft-карточки и источники |
| `graph_gap_finder` | запросы про граф, prerequisites, пробелы, “что мешает понять тему” | читает профиль, graph/mastery и при необходимости RAG, затем объясняет пробелы и порядок закрытия prerequisites |
| `living_konspekt_coach` | запросы про `konspekt`, workbench, корзину или Живой конспект | читает текущую корзину конспекта и предлагает, что добавить, что повторить, как проверить понимание и какие draft-карточки сделать |

Пример API-запроса:

```json
{
  "question": "Собери короткую учебную сессию по правилу Байеса",
  "query_mode": "agent",
  "session_id": "study-agent-demo"
}
```

В ответе `debug.answer_path.scenario_id` показывает выбранный сценарий,
`debug.agent_trace.run_id` — идентификатор сохранённого прогона, а
`debug.agent_trace.tool_calls` — какие read-only инструменты были вызваны.
Публичный `answer_status` остаётся совместимым с `/ask`:
`grounded`, `abstain` или `guardrails_fallback`.

При переходе на «Агент» с Mission Control (или из других мест, где установлена `current_topic`) поле ввода автоматически предзаполняется текущей темой/курсом (A1 Polish).

После получения ответа в секции «## Карточки-кандидаты» появляются кнопки «Сохранить» рядом с каждым кандидатом. Нажатие создаёт колоду «Агентские черновики — ...» и добавляет карточку через `add_flashcard` (B2). Это user-initiated сохранение предложенных агентом кандидатов (агент сам ничего не пишет в БД).

История прогонов агента видна студенту в разделе «Прогресс обучения» (вкладка/экспандер «Что агент собирал для вас»). Показываются последние запуски с вопросом, статусом и коротким `run_id` (C1, данные через `GET /agent/runs`).

Для команды доступен read-only API `/agent/runs` и `/agent/runs/{run_id}` (защищённый).

## Первые 10 минут (activation)

После «Начать» на welcome можно включить подсказки **«первые 10 минут»**:
короткая лента рядом с экраном (не блокирующий tour). Шаги идут по **реальным
действиям** (курс → вопрос → источник → тьютор → quiz → память → главная).
Пропустить шаг / выйти можно в любой момент; полный справочный тур (несколько
глав) запускается только вручную с главной («Справочный тур»).

## Quiz

Quiz поддерживает генерацию и оценку вопросов по теме или документу.

**Интерактивный Quiz** (раздел UI): персональные вопросы с типами «выбор»,
«верно/неверно», «пропуск», «порядок». Разбор и правильный ответ показываются
**только после** кнопки «Ответить» (нельзя подсмотреть до фиксации). Порядок
пунтов — кнопками ↑/↓ без drag. Обновление графа и сохранение в историю —
только после «Завершить quiz…» и хотя бы одного зафиксированного ответа.
Celebration (balloons) — при результате ≥80%; при низком score — спокойный
recovery (flashcards), без праздника.

Ключевые endpoints:

- `POST /quiz/generate`
- `POST /quiz/evaluate`

Оценка возвращает diagnostic feedback: `recognized`, `recalled`, `misconception`, `cannot_apply`. Эти сигналы участвуют в учебном контуре и могут влиять на следующий шаг.

## Flashcards и SM-2

Flashcards — это human-in-the-loop генерация:

1. выберите источник: документ, upload или active course;
2. сгенерируйте preview;
3. отредактируйте карточки;
4. сохраните колоду;
5. повторяйте по SM-2.

Поддерживается:

- due-очередь;
- оценки `Again / Hard / Good / Easy`;
- recovery tail для большой очереди;
- undo recovery для ещё не повторённых карточек;
- экспорт Anki `.apkg`;
- импорт карточек из quiz.

Ключевые endpoints: `/flashcards/*`.

## Knowledge Graph: точные разделы лекций

Раздел `Knowledge Graph` разделён на два явных таба. **«🕸 Граф знаний»**
открывается по умолчанию и содержит 2D-карту, действия с концептом и диагностику.
**«🌆 Мнемополис»** содержит Memory Run и встроенный 3D-зал; боковая кнопка
«В Мнемополис» сразу выбирает этот таб.

Статус карты на табе «Граф знаний» и в подготовке курса написан учебным
языком: «Карта актуальна» / «Показана предыдущая версия карты — новая ещё не
готова» / «Карта знаний пока не собрана…». Технические идентификаторы (если
нужны) спрятаны в «Технические детали карты». Если свежая карта не прошла
проверку качества, показывается черновик только для диагностики — он не
используется в ответах и плане; в раскрытии видны причины сбоя. Если
единственный блокер — мало документов, текст формулируется как лестница:
«Добавьте ещё N документ(ов) курса — появится семантическая карта».

Во вкладке **Темы → подготовка активного курса** есть **«Паспорт материала»** —
одна карточка из уже посчитанных сигналов: свежесть/публикация карты, покрытие
конспектами, доля media-sidecar, готовность корпуса и аудит дубликатов концептов.
Новых вычислений retrieval/LLM нет: только сведение сайдкаров.

Если у конспекта в frontmatter есть `source_sha256` (Obsidian export / smart-konspekt)
и исходная лекция после этого изменилась, у документа появляется бейдж
**«🕰 устарел»** — без LLM, сравнение хэша/mtime. Конспекты без `source_sha256`
бейдж не показывают (неизвестно).

Граф знаний — не только карта концептов. У каждого концепта в панели документов показываются **точные разделы** конспекта, где он разобран (до трёх мест: тема, антипаттерны, термины):

- **«⬇ Скачать живую карту (HTML)»** сохраняет текущую интерактивную карту курса одним
  файлом `knowledge_graph.html`: узлы, зум, клики и локальный D3-рендер остаются доступны
  при открытии файла вне приложения;
- **«⬇ Скачать 3D-зал (HTML)»** (экспорт) — самодостаточный 3D-визуализатор (canvas, без CDN)
  в скине **Memory Run** (U5 HUD): топбар (бренд, сегменты, прогресс-кольцо) и два
  плавающих дока поверх сцены во всю ширину — **карточка остановки слева** (`#stopdock`),
  **«Маршрут дня» справа** (`#side`); каждый док скроллится сам, маршрут виден сразу
  без nested-scroll колонки; камера учитывает insets (`--hud-l`/`--hud-r`).
  На узких экранах доки складываются под сценой (☰ скрывает обе панели).
  **Первый кадр — маршрут дня**, не весь граф:
  остановки `day_route` (тот же `select_day_route`, что и 2D «Авто: маршрут дня»),
  активный стоп, причина (`worth_reason`) и подписи с лимитом (≥12px + halo).
  В режиме **Маршрут** полный граф скрыт: остановки и тихие lesson-якоря; контекст —
  в **Созвездии** (production `local`); **Вся карта** — углубление из Созвездия.
  **След памяти** — оверлей поверх любого режима (не отдельная навигация).
  **Экскурсия дня** — управляемый пролёт (← / ▶ / → на сцене), не таймер-скачки.
  Геометрия: дорожка `1 → N`; в созвездии/карте **Y** = этаж урока (`precedes`),
  **X** = прогрессия, **Z** = полоса; **worth не двигает высоту** — ранг, карточка
  (роль, W, mastery-кольцо, due/novel, дата квиза). **✓** = был в квизе
  (`mastery_history`, любой score; ≥80% ярче). **◆** / двери Obsidian — только
  embedded. **Рассвет и фонари:** небо и огни на остановках кодируют quiz-след
  маршрута дня (0/N — ночь, N/N — рассвет; без отдельной валюты).
  **Туман забывания** (◌ «Спокойный мир» в топбаре): дымка по низкому retention
  из `decay_vector` (сила = 1 − retention). В режиме «Маршрут» — тихие маркеры;
  полная дымка — в Созвездии/Всей карте. Туман **не блокирует** «Начать» /
  «В конспект»; полное имя концепта всегда в карточке.
  **🔁 Повторить** (embedded): при тумане или due открывает Flashcards (раздел
  повторения), **автозагружает due-очередь** с фокусом концепта (теги + фильтр
  front/back) — retrieval practice «развеять туман», без записи в домен. **Хранитель Мнемо** (W3a–W3d): в карточке — «экскурсия»
  (worth_reason) и «угрозы забывания» (из `decay_vector` + due); в панели маршрута —
  **цель утра** (квестмейстер: degrade «N из M», без валюты);
  **голоса** антагонистов (статика/кэш, без стыда);
  **летопись** у G4.2-реплея (quiz-снимки);
  **призрак уверенности** — полупрозрачный ✓ при «знал, но забывает» (local/all; ◌ скрывает);
  **разлом** — трещина при слабых *conceptual* prereqs (`node.prereqs`, не lesson-edges; CTA не блокирует);
  **стройка** — баннер только если карта honestly stale/не готова (publish status);
  **📷** — G4.3 локальный PNG (скачивание в браузере, **без** сервера/облака);
  **пресеты сцены** + **NL-команды** («слабое», «маршрут», «фокус RAG», «сброс») —
  presentation-only (validator + deterministic parse, без eval/JS);
  `day_route` домена не переписывается (W5b.1/W5b.2).
  Хранитель: офлайн/LLM для экскурсии, угроз, цели, **голосов** и **летописи**.
  🔁 — развеять туман. Export:
  **«снимок от YYYY-MM-DD»**, CTA read-only («… · в продукте»), без корзины и
  vault-ссылок. Подписи ≤8; JSON в HTML script-safe.
- **🌆 В Мнемополис** (сайдбар) — deep link в раздел Knowledge Graph на таб «Мнемополис»
  (город памяти). **Не** заменяет Mission Control: home остаётся главным экраном.
- **🌆 Вернуться в Мнемополис** — после quiz / после сессии Flashcards review:
  честные каналы (quiz → ✓/рассвет; SR → туман; collect ◆ уже в зале при «В конспект»).
- **Районы (двери)** в режимах Созвездие / Вся карта: ⚔ Арена → Quiz,
  🌿 Оранжерея → Flashcards, 🛤 Магистраль → Адаптивный план, 🔨 Кузница →
  Живой конспект. В первом кадре Маршрута дверей нет (ориентация).
- **📜 Кратко** (карточка / интерьер) — read-only справка из данных графа
  (mastery, описание, опоры, имена источников); остаётесь в 3D-зале, без LLM.
- **💬 Спросить** (карточка / интерьер узла) — handoff в «Чат с тьютором» с
  префиллом по концепту (полный разговор вне зала).
- **🏛 3D-зал (embedded)** на табе «Мнемополис» — живой Memory Run, отделённый от 2D-карты.
  **Каналы действий (надёжность iframe):** primary — `setComponentValue` с envelope
  `{kind: 'kg3d_action', envelope}` (работает внутри Streamlit component iframe);
  fallback/compat — query-param `_kg3d` (base64url + `session_nonce`), если
  `top.location` доступен. Selection — строковый concept id через тот же
  `setComponentValue`. Оба канала делят дедуп `event_id` (окно 64) — dual delivery
  не дублирует side effect. **▶ Начать** → «Интерактивный Quiz» с преселектом
  концепта; **В конспект** — тот же collect, что 2D; **Открыть раздел** / интерьер
  («Внутрь» или повторный клик) — разделы с `obsidian://`. **Правила** в топбаре
  открывает правила зала; автопоказ не повторяется после подтверждения и рестарта браузера.
  **◆** и счётчик корзины — live view-model. Export read-only.
- **«📍 «заголовок»»** открывает Obsidian сразу на нужном заголовке конспекта (нужен созданный конспект — кнопка «Подготовить для Obsidian» во вкладке `Темы`);
- **«🖥 VS Code: раздел»** открывает тот же файл на нужной строке;
- при повторяющихся заголовках в документе UI подсказывает, что VS Code-ссылка точнее (Obsidian-якорь открывает первый одноимённый заголовок);
- для `.txt` без конспекта вместо мёртвой кнопки — подсказка подготовить конспект.
- Граф знаний показывает связи, прогресс и пробелы, но не отдельный weekly-план. «План на сегодня» строится адаптивно, и это единственный источник для продолжения обучения.

В панели «⚡ Действия с концептом»:

- **«🔬 Качество графа»** показывает score, orphan-узлы, duplicate/alias suspects,
  концепты без документов/точных разделов, связи без полного evidence и
  test/source artifacts, попавшие в граф;
- **«➕ раздел «заголовок»»** (до трёх на документ) кладёт раздел в корзину «Живого конспекта»;
- **«➕ Собрать всё по концепту»** добавляет лучший раздел каждого связанного документа одним кликом;
- **«📚 Живой конспект (N)»** показывает счётчик корзины и ведёт в режим сборки.
- **«🧾 Почему этот узел есть»** показывает local evidence ledger: описание узла,
  aliases, prerequisites, связанные документы и найденные точные разделы;
- **«🧬 Возможные aliases / дубли»** подсвечивает deterministic кандидаты на merge
  по пересечению label/aliases. Это диагностический сигнал: граф не меняется сам.

Такие же ссылки появляются под карточкой во Flashcards: «Открыть раздел» (Obsidian/VS Code) и «➕ В рабочий конспект» — карточка ведёт в точное место лекции, из которого она родилась.

## Живой конспект

Режим `Живой конспект` собирает выбранные разделы разных лекций в рабочий конспект — не теряя контекста и «духа» исходных лекций.

**Корзина разделов** наполняется из графа знаний, из Flashcards и прямо из режима `Живой конспект`: панель «📥 Добавить разделы» ищет по markdown-конспектам в `data/` и позволяет открыть документ целиком, добавить отдельный раздел или все содержательные H2-разделы. Дедупликация — по файлу и строке. Корзина **автосохраняется** в локальный профиль (`app_kv` в `data/user_state.db`) при каждом изменении и переживает перезапуск приложения; на Mission Control появляется resume-карточка «Живой конспект — сборка не закончена» с кнопкой «Продолжить сборку». Именованные research-сессии в сайдбаре («Исследования») остаются как снимки-варианты корзины.

Рабочая поверхность разделена на вкладки:

- **🧩 Разделы** — управление корзиной, bulk-действия «убрать документ», явная очистка одного/нескольких разделов или всей корзины с подтверждением, deep-link'и в Obsidian/VS Code, плейлист «Мои N минут», медиа раздела, аудио-подкасты и кнопки `↑`/`↓` для порядка будущего конспекта; если исходный конспект изменился, переехал или удалён, под разделом появляется подсказка «🕰 …» (корзина продолжает работать со снимком). На вкладке тем и в карточках теперь видно «рубрика X/5» для cloud-конспектов (A1 паспорт качества).
- **📖 Читать** — полный собранный конспект прямо в приложении: TOC для длинных сборок, главные мысли исходных лекций, тексты разделов по порядку, медиа, заметка «Моя мысль», «Мой вопрос», кнопки статуса (Понял / Сомневаюсь / Не понял + «Прочитано»), отметка «Прочитано» и «Проверь себя». В caption источника и бейджах — рубрика качества если есть.
- **🧠 Память** — due-карточки по `source:`-тегам, создание карточек из раздела терминов и scoped quiz по текущей сборке;
- **📚 Сохранить** — дословная сшивка или LLM-синтез в `data/living-konspekt/`;
- **🌐 Дальше** — ссылки для проверки актуальности, graph-lens по связанным концептам, coverage map активного курса и deep-study prompt.

## Аудио-подкасты (Audio Podcasts)

Позволяет слушать лекции «в ушах» прямо из Живого конспекта — без переключения на внешний плеер.

### Как получить аудио

Оффлайн-шаг пайплайна извлекает аудиодорожку из видео (только копирование, без перекодирования):

```powershell
# Полный пайплайн (рекомендуется)
.\scripts\Run-MediaKonspektPipeline.ps1 `
  -Konspekt "ИИ Агенты/урок_2_....md" `
  -Media "D:\AI\app\data\ИИ Агенты\урок_2_....mp4" `
  -SkipAsr

# Или отдельно после транскрибации
.\scripts\transcribe_media.py "video.mp4" --remux
```

В результате рядом с `.mp4` появляется sibling-файл `<video>.m4a`.

### Требования

- `ffmpeg` в `PATH` (для извлечения `.m4a` и склейки выпуска).
- Установка: `winget install Gyan.FFmpeg` (Windows) или эквивалент.

Без ffmpeg или без `.m4a` — полная деградация к предыдущему поведению (видео и подписи остаются).

### В интерфейсе Живого конспекта

- **Разделы**: если есть `.m4a` и доверенный таймкод — появляется чекбокс «🎧 Слушать раздел». При активации рендерится `st.audio` с обрезкой `start_time` / `end_time` (формат `audio/mp4`).
- **Плейлист «🎧 Мои N минут»**: локальные фрагменты с аудио становятся интерактивными (чекбоксы → последовательность плееров). YouTube-фрагменты остаются ссылками.
- **Скачивание выпуска**: кнопка «⬇️ Скачать выпуск (m4a)» рядом с плейлистом собирает все доверенные аудио-фрагменты корзины в один файл (`ffmpeg` concat без перекодирования) + текстовое оглавление с таймкодами (время в готовом файле начинается с 0:00).

Собранный выпуск можно сразу слушать на телефоне или сохранить «в дорогу».

### Технические детали

- Обнаружение `.m4a` — по конвенции имени файла (рядом с видео из `media_sidecar`), без изменения схемы sidecar v1.
- Таймкоды берутся из доверенных разделов `media_sidecar` (только `t_start` + `t_end`).
- Реализация: `app/media_audio.py` (discovery + `make_basket_audio_release`), рендер в `app/ui/living_konspekt_media.py`.
- Пайплайн всегда пытается извлечь аудио после sidecar (см. `Run-MediaKonspektPipeline.ps1`).

Текущий статус: P0 (звук + выпуск) + P1 (B1 listen trace в learner state как `listened_at`, B2 TTS-файлы из текста через [tts], B3 унификация ASR на faster-whisper local-first с удалением Google).

- Статус прослушивания (`listened_at`) сохраняется вместе с `read_at`/`note`, переживает перезапуск и «Пересобрать».
- TTS: генерация аудио-файла из текста раздела/транскрипта (демо в сайдбаре «Голос»).
- Голосовой ввод (sidebar + TG заметки) теперь использует локальный Whisper при наличии `[asr]`.

Доставка в Telegram, диалоговые подкасты (opt-in) и «сегмент дня» — кандидаты P2.

**🎞 Все видео урока** появляется над собранными разделами, если хотя бы один документ
корзины связан с валидным `media_sidecar`. Панель показывает полный список `media.videos[]`
по документу: каждое видео — в отдельном схлопываемом блоке; локальное видео как встроенный
плеер, YouTube — как встроенный iframe-плеер с дополнительной ссылкой «Открыть на YouTube»,
прочие внешние URL — как безопасные ссылки.

**📥 Быстро добавить разделы** внутри вкладки «🧩 Разделы» позволяет выбрать документ из текущей корзины и добавить в неё все крупные H2-разделы этого документа одним действием. Общая панель «📥 Добавить разделы» над вкладками работает и на пустой корзине: это основной вход, когда вы хотите собрать конспект без предварительного перехода через Knowledge Graph.

**🎧 Мои N минут** появляется, когда у собранных разделов есть доверенные таймкоды в
`media_sidecar`. Это плейлист фрагментов в порядке вашей корзины: устаревшие или
low-confidence таймкоды не попадают в список.

Подробности использования аудио (прослушивание разделов, интерактивный плейлист и скачивание выпуска) — в подразделе **Аудио-подкасты (Audio Podcasts)** выше.

**🎬 Материал раздела** появляется внутри карточки собранного раздела, если у исходного
конспекта во frontmatter есть валидный `media_sidecar`, sidecar содержит matching section
timestamp, а confidence таймкода достаточный. `media.video` остаётся основным видео, а
опциональный список `media.videos[]` показывает ролики урока в общей панели. Для YouTube
показывается встроенный плеер; при уверенном таймкоде воспроизведение стартует с нужной
секунды, виден диапазон раздела (`M:SS–M:SS`) и рядом остаётся ссылка «Открыть на YouTube
с M:SS». Для локального видео файл рендерится только если путь в sidecar остаётся внутри
`data/`; абсолютные и traversal-пути отклоняются. Если sidecar устарел или confidence
таймкода низкий, UI показывает degraded state; low-confidence timestamp остаётся
«видео-кандидатом» и не встраивает видео в раздел.

Аудио-функции (включая `st.audio` и скачивание выпуска) подробно описаны в подразделе
**Аудио-подкасты (Audio Podcasts)** выше.

**Схема раздела** внутри раскрытого содержимого раздела генерирует Mermaid-схему по тексту
этого раздела через graph LLM. Результат сначала показывается как предпросмотр; markdown
исходного конспекта меняется только после явного «Принять схему». «Отклонить» очищает
preview без записи.

Во вкладке **📖 Читать** можно оставить заметку «Моя мысль» под каждым разделом и
отметить его прочитанным. Эти поля (`note`/`read_at`) сохраняются в локальной корзине,
переживают пересборку сохранённого артефакта и при экспорте выводятся блоком
«💬 Моими словами» под соответствующим разделом.

Поле **«Цель конспекта»** хранится рядом с корзиной, сохраняется в manifest артефакта
и используется как контекст для проверки себя. Во вкладке **🧠 Память** кнопка
«Сгенерировать 6 вопросов по моей сборке» строит quiz строго из выбранных строк
Живого конспекта, без повторного retrieval по индексу.

Текущий статус мультимодальности: ручной/внешний sidecar уже читается, но импорт видео, ASR,
автоматическое выравнивание таймкодов, VLM captions и `media_progress` ещё не являются
пользовательским flow.

**📚 Собрать рабочий конспект** — два способа:

- *Дословная сшивка (без LLM)*: разделы склеиваются как есть; в шапку добавляется главная мысль каждой исходной лекции;
- *LLM-синтез из разделов*: модель получает именно выбранные тексты разделов (не повторный поиск по индексу).

В обоих режимах файл заканчивается блоками Study Pack: «✅ Проверь себя» (контрольные вопросы лектора, дословно), «🎬 Видео материалов» (все видео-источники из media-sidecar'ов корзины — первичная лекция и смежные материалы, URL кликабельны) и «Источники» (`файл:строки` каждого раздела). При дословной сшивке под источником раздела добавляется его видео-таймкод (`*🎬 … · 12:34–18:02*`, для YouTube — кликабельный) — только если sidecar актуален и confidence раздела достаточен: недоверенный таймкод в файл не пишется. Результат сохраняется в `data/living-konspekt/*.md` и попадает в поиск и граф после обновления индекса (UI честно сообщает об этом); под кнопкой сборки остаются deep-link'и на последний собранный файл (Obsidian/VS Code).

**🧠 Память конспекта** — обратная петля повторения: если карточки, привязанные к конспектам корзины (`source:`-тег), стали due по SM-2, панель показывает «файл — N к повторению», а кнопка «🔁 Повторить» открывает Flashcards Review с фильтром именно по этому конспекту; дальше нажмите «Загрузить очередь». Конспект сам подсказывает, что вы начали забывать, и возвращает к точному месту. Пока всё повторено — панель молчит.

**🃏 Карточки из терминов лекции (без нового LLM-вызова)** — если конспекты собранных разделов содержат раздел «🧠 Важные термины и концепции» (формат `- **Термин** — определение.`), кнопка «Создать карточки из терминов» детерминированно парсит сохранённые пары термин/определение в колоду. Карточки открываются в готовом preview-редакторе Flashcards (вкладка «✨ Создать новые») — отредактируйте или удалите лишнее и сохраните как обычную колоду с SM-2 повторением. Для сохранения нужно минимум 5 карточек, как и в обычном редакторе Flashcards.

**🌐 Проверить актуальность · источники** — local-first: приложение в сеть не ходит, вы получаете готовые ссылки:

- **«🔗 Ссылки из лекции»** — источники, которые лектор сам приложил к материалу (из текстов разделов и раздела «Дополнительные материалы» конспекта);
- кнопки поисковиков (Google, Scholar, arXiv, Perplexity, YouTube) с коротким запросом из концептов и ключевых слов заголовков.

**🕸 Граф-линза** показывает, что граф знаний видит рядом с текущей сборкой: недостающие
prerequisites и соседние темы по концептам, из которых были добавлены разделы.

**🗺 Покрытие курса** появляется, когда активирован course scope: панель сравнивает
документы курса с источниками текущей корзины и показывает, какие лекции уже покрыты
рабочим конспектом, а какие ещё стоит добавить.

**🧠 Промпт для глубокого изучения** — копируемый промпт для облачной модели (ChatGPT/Claude/Gemini): главная мысль лекции, дословные цитаты выбранных разделов с источниками, prerequisites/связанные концепты, а также риски/антипаттерны и контрольные вопросы из исходного конспекта. Формулировка требует верности рабочему конспекту и проверки актуальности с источниками.

## Темы и курс

Раздел `Темы` показывает каталог материалов, synthesis и программу обучения («Программа обучения» — долгосрочный табличный план курса; «План на сегодня» в Mission Control — адаптивный daily маршрут). Если материалы организованы папками, активируйте папку как course scope и работайте с ней как с курсом. Когда активный курс сводится к одной теме, рабочая область документов раскрывается на всю ширину без лишней карты тем; встроенное чтение подготовленного конспекта тоже открывается отдельной широкой панелью. Технические и тестовые папки (`_test*`, cache, graph artifacts) в выборе курса скрываются.

Табличная программа обучения читается как набор шагов по строкам таблицы. На
экране она показывается как компактные строки с прогрессом, практикой,
проверкой результата и кнопками материалов; экспорт в Markdown сохраняет
табличный вид. При наличии подготовленных markdown-конспектов строки программы
получают inline-ссылки на раздел: Obsidian, VS Code и доверенный видео-таймкод
из `media_sidecar`, если он есть и не устарел. Если точный раздел ещё не
подготовлен, строка всё равно даёт ссылку на исходный документ. Если сумма значений в колонке «Время (ч)» превышает
заданный бюджет, UI показывает предупреждение и предлагает сократить шаги или
пересобрать программу.
Если для активного курса уже есть сохранённая программа, кнопка «Открыть кэш»
загружает её без LLM-вызовов, а «Пересобрать программу» заново строит обзор и
программу курса по текущим документам и сохраняет результат поверх кэша.
В сайдбаре активного курса есть кнопка «Открыть программу курса», чтобы вернуться к
сохранённой программе без поиска по вкладкам.
После сохранения шага программы на главном экране появляется карточка «Продолжить
тему или программу» с указанием шага, на котором остановились. Кнопка «Продолжить
обучение» в сайдбаре также учитывает сохранённый шаг программы.
Флажок «Учитывать прогресс» в форме программы автоматически включён, если карта
знаний (knowledge graph) содержит концепты — граф и прогресс влияют на
подсказку для порядка шагов. Отключите флажок для свободной (free-form) программы.

Course Cockpit показывает daily briefing, pace mode, активность курса и прогресс до graduation.
Активный курс сохраняется локально и переживает перезапуск приложения: после старта
он снова активен без повторного выбора папки (если папка курса на месте).
Если курс временно деактивирован, его можно восстановить из сайдбара, Mission Control
или списка тем без повторного выбора папки — последний деактивированный курс тоже
переживает перезапуск.

## Прогресс обучения

Раздел `Прогресс обучения` отвечает на вопрос “что делать сегодня?”.

Там собраны:

- adaptive daily plan;
- mastery;
- analytics;
- weekly narrative;
- due/review состояние;
- Knowledge Graph и weak spots.

Единый источник “сегодняшнего” шага — Adaptive Daily Plan: тот же primary block
показывается на Mission Control и используется Telegram-напоминанием. AI Coach и
weekly narrative остаются аналитикой на несколько дней; граф знаний не показывает
отдельный weekly-план как маршрут действий. Если ежедневный снимок ещё не
сформирован, Telegram показывает общую подсказку открыть приложение для
персонального плана.

Локальное состояние хранится в `data/user_state.db`.

## Smart Study Router

Smart Study Router выбирает следующий шаг из локальных сигналов:

- due flashcards;
- concept-level SM-2;
- свежий ответ с источниками;
- ошибка в quiz;
- weak concept;
- tutor resume;
- active course/adaptive plan.

Результат: `hint_kind`, primary action, secondary actions и объяснение. Feedback `accept/reject/defer` сохраняется локально через `/ssr/recommendation-feedback`.

### Что видно в блоке «Как выбрана подсказка»

Свернутый по умолчанию блок «Как выбрана подсказка» на Mission Control раскрывается по клику. Внутри:

- **Видны всем** — человекочитаемые секции: «Другие варианты», «Если выбрать иначе», «Маршрут». Они написаны обычным языком и объясняют выбор шага без технического жаргона.
- **Только экспертный режим (tier 5 / Debug)** — секция «Локальные сигналы» с сырой диагностикой SSR (очереди flashcards/SM-2, source-trust, локальный руль, ml-audit-trace). Обычный пользователь её не видит, даже раскрыв блок. Включается через настройки уровня интерфейса (уровень 5) вместе с панелью «Debug: маршрутизация, trace, стоимость» (`panel:debug_summary`).

## Приватность и данные

| Данные | Где |
|---|---|
| исходные материалы | `data/` |
| собранные рабочие конспекты («Живой конспект») | `data/living-konspekt/` |
| прогресс, flashcards, mastery | `data/user_state.db` |
| индекс | `chroma_db/` |
| логи и SSR-профили | `logs/` |
| настройки по умолчанию | `config.env` |
| локальные секреты | `.env` |

По умолчанию `config.env` использует локальные LLM/embedding endpoints. Чтобы ни один фрагмент не покидал вашу машину, держите `EMBED_API_BASE`, `EMBED_MODEL`, `LLM_API_BASE` и `LLM_MODEL` направленными на локальные серверы. При использовании облачного LLM/embedding provider содержимое запросов уходит внешнему провайдеру.
