# Мультимодальный конспект: анализ, US/CJM и план «из скучного конспекта — шедевр»

Статус: ТЗ-проработка (handoff-документ, самодостаточный).
Подготовлено: 2026-07-05. Родственные документы: `docs/first_run_experience_plan.md` (паттерн handoff-плана),
`doc/archive/strategic_review_2026-07-03.md` (стратегические ограничения — учтены в разделе 8).

---

## 1. Суть в трёх абзацах

hometutor сегодня — текстовый продукт: `.txt/.md/.pdf/.html/.docx` на входе, markdown-конспект на выходе.
При этом реальный учебный материал студента — это видеолекции, слайды и диаграммы. Цель — сделать
«Живой конспект», обычные конспекты и активный курс мультимодальными: видео с таймкодами по разделам,
слайды и картинки внутри разделов, интерактивные вопросы прямо в просмотре.

Главная архитектурная находка анализа: **швы для мультимодальности уже существуют**.
`smart_konspekt.gather_lecture_inputs()` уже классифицирует папку лекции на транскрипты/черновики/презентации —
видео входит туда через один шаг «video → ASR → transcript.txt» без изменения контракта. Section Anchor Index
уже делает каждый раздел конспекта адресуемым (slug, диапазон строк, deep-links) — это готовая точка крепления
таймкодов и медиа. «Живой конспект» уже собирает и рендерит ссылки, включая YouTube-поиск (`study_web_queries.py`).

Главный дефицит — не в архитектуре, а в парке моделей: в `local_models_comparison_current_v3.md` нет ни одной
ASR- и ни одной vision-модели. Железо (2× RTX 5070 Ti 16 GB) покрывает обе роли с большим запасом.

## 2. Анализ готовности архитектуры

### 2.1 Что уже готово (швы для встраивания)

| Шов | Где | Что даёт мультимодальности |
|---|---|---|
| Контракт входов лекции | `app/smart_konspekt.py::gather_lecture_inputs` — transcripts (`.txt`), drafts (`.md`), html_notes, presentations (`.pdf`) + бюджеты по типам в Settings | Транскрипт видео = ещё один `.txt` в `transcripts`. **Ноль изменений контракта.** |
| Section Anchor Index | `app/section_index.py` — `ParsedSection`/`IndexedSection`: slug, `line_start/end`, `own_text`, скоринг `tokenize_filtered` | Готовая адресация разделов → к разделу крепятся таймкоды видео, слайды, картинки. Скоринг `own_text` переиспользуется для выравнивания «раздел ↔ сегмент транскрипта». |
| Живой конспект | `app/ui/living_konspekt_view.py` — корзина разделов, автоперсист в `app_kv`, `study_web_queries.py` уже строит YouTube-ссылки и собирает ссылки из тел разделов | Медиа-панель раздела — естественное расширение существующей ссылочной механики. |
| Obsidian-экспорт | `app/obsidian_export.py` — YAML-frontmatter, sha256-кэш, map-reduce compose | Frontmatter несёт `video_source`/`media`; map-reduce уже решает проблему длинных транскриптов при 32K контексте. |
| Граница провайдера | `app/provider.py` — все LLM/embedding клиенты через одну фабрику; OpenAI-совместимый протокол | VLM через LM Studio — это тот же OpenAI-совместимый чат с `image_url`. Vision добавляется **внутри существующего правила**, не ломая гвард. |
| Docling-флаг | `app/config.py::ingest_docling_enabled` (US-2.3 phase1: сканы/изображения) | Путь «картинки в общий индекс» уже спроектирован, просто выключен. |
| Учебный контур | quiz → `flashcard_handoff` → SM-2 → mastery → SSR | In-video вопросы падают в **уже существующий** конвейер «ошибка → карточка → повторение». |
| Реестр фич UI | `app/ui/feature_registry.py`, 4 tier'а | Медиа-фичи входят на tier 2–3, не перегружая новичка. |
| Streamlit UI | `st.video(start_time=…)`, `st.image` — нативно | Рендер медиа почти бесплатен. |

### 2.2 Чего нет (дефициты)

| Дефицит | Последствие | Сложность закрытия |
|---|---|---|
| Нет ASR в контуре (ни модели, ни ffmpeg) | Видео — «слепой» формат | S–M: скрипт + optional extra |
| Нет vision-модели в парке | Слайды/диаграммы не читаются | S: LM Studio + Qwen3-VL, через `provider.py` |
| Нет медиа-полей в данных | Section index / chunk metadata / user_state не знают про таймкоды и картинки | M: sidecar-файлы + одна таблица через `user_state*` |
| Конспект рендерится как чистый markdown | `st.video/st.image` используются только в сайдбаре | S: медиа-панель раздела |
| Quiz/flashcards текстовые | Нет вопросов с таймкодом, нет image-карточек | M: расширение генерации |
| Контекст Qwopus 32K | Транскрипт >1.5 ч не влезает целиком | Уже решено: map-reduce в compose |

### 2.3 Ограничение сверху

Стратегический обзор 2026-07-03 требует «прекратить наращивать поверхность». Мультимодальность проходит
фильтр «углубляет ли это цикл ответ→tutor→quiz→cards→progress→SSR?» — **да, углубляет каждый узел**
(конспект богаче, quiz живее, карточки визуальнее, SSR получает новый сигнал). Но правило дисциплины:
новые модули маленькие, зависимости — в optional extras, флаги — off по умолчанию, каждая фаза даёт
самостоятельную ценность и может быть последней.

## 3. Транскрибация видео: оптимальное решение

### 3.1 Сравнение вариантов (локально, RU+EN лекции, 2× RTX 5070 Ti)

| Вариант | Скорость | RU качество | Таймкоды | Интеграция | Вердикт |
|---|---|---|---|---|---|
| **faster-whisper (large-v3 / turbo)** | ~4× быстрее оригинала на GPU; часовая лекция — минуты | Отличное (99 языков, лучший выбор для RU/смешанных RU-EN лекций с терминами) | Word-level + VAD из коробки | pip-пакет, python-API — идеально для скрипта в `scripts/` | ✅ **Рекомендация** |
| whisper.cpp | ~10× real-time и выше | То же ядро (large-v3) | Есть | Внешний бинарь, без python-зависимостей | Запасной путь, если не хочется CUDA-python-стека |
| NVIDIA Parakeet TDT v3 | Экстремальная (~2000× RTFx), 6.32% WER против 7.44% у whisper | RU входит в 25 языков, но модель ориентирована на европейский набор; на смешанных RU-EN лекциях с кодом/терминами whisper надёжнее | Есть | NeMo-стек тяжелее | Кандидат на будущее, когда объёмы вырастут |
| Cloud API (OpenAI/Gladia) | Быстро | Хорошее | Есть | Против local-first ДНК проекта | ❌ Только как явный opt-in |

### 3.2 Рекомендуемый конвейер

```text
video.mp4
  → ffmpeg: extract audio (16 kHz mono wav)          [ffmpeg — системная зависимость, проверка в preflight]
  → faster-whisper large-v3 (CUDA, VAD, word timestamps)
  → два артефакта рядом с видео в папке лекции:
      lecture.txt            — чистый транскрипт → подхватывается gather_lecture_inputs() КАК ЕСТЬ
      lecture.segments.json  — [{start, end, text}] → выравнивание разделов конспекта по времени
  → smart_konspekt compose (существующий map-reduce, Qwopus36)
  → конспект с таймкодами разделов (frontmatter: video_source + section_timestamps)
```

Пример для файла из задачи:

```powershell
# Фаза 1, scripts/transcribe_media.py (новый, ~150 строк):
.\.venv\Scripts\python.exe scripts\transcribe_media.py `
  "C:\Users\Kostya\Downloads\The_Architecture_of_Autonomy.mp4" `
  --lecture-dir "materials\autonomy\lecture_01" --language auto --model large-v3
# → materials/autonomy/lecture_01/The_Architecture_of_Autonomy.txt
# → materials/autonomy/lecture_01/The_Architecture_of_Autonomy.segments.json
# Дальше обычный smart-konspekt путь без изменений.
```

Ключевые свойства: идемпотентность по sha256 медиафайла (паттерн уже есть в `obsidian_export`),
extra `asr` в `pyproject.toml` (`faster-whisper`), полностью offline.

### 3.3 Выравнивание «раздел ↔ таймкод»

Два уровня, от простого к точному:

1. **Compose-время (дёшево):** в map-фазе конспекта каждому chunk'у транскрипта передаётся его временное окно;
   промпт просит сохранять маркер `[t=MM:SS]` у заголовков. Compose уже проходит через `app/prompts/` — правка промпта.
2. **Post-hoc (точно, без LLM):** скоринг `own_text` раздела против окон `segments.json` тем же
   `tokenize_filtered`-механизмом, что уже используется в `section_index.py` для матчинга. Детерминированно, тестируемо.

Результат хранится sidecar'ом `<konspekt>.media.json` (не ломает существующий парсинг md) —
`{slug: {t_start, t_end, images: [...]}}`.

### 3.4 Минимальный путь (MVP-0, без ASR вообще)

Если видео лежит на YouTube или транскрибация пока не нужна: поле `video: <url|path>` в frontmatter конспекта
+ рендер в «Живом конспекте» (`st.video` / ссылка с `?t=`). Один день работы, ноль зависимостей,
немедленная ценность. Это ответ на «или хотя бы добавления ссылки» — и это правильная первая фаза, а не компромисс.

### 3.5 Vision-слой (слайды и картинки)

- **Модель:** Qwen3-VL (сильнейшая открытая VLM: OCR, диаграммы, таблицы, видеокадры) через LM Studio —
  OpenAI-совместимый endpoint → новая фабрика `get_vision_llm()` в `app/provider.py`, правило границы соблюдено.
- **Источники картинок:** страницы PDF-презентаций (`pymupdf` уже в optional extras!) и/или кейфреймы видео
  (ffmpeg scene detection на границах сегментов).
- **Продукт:** подпись/alt-text каждого слайда → в тело раздела конспекта (поиск по подписям работает через
  существующий BM25/vector индекс — **мультимодальный retrieval без мультимодальных эмбеддингов**).
- Qwen3-VL-Embedding (единое пространство текст+картинки) — отметить как «наблюдать, не строить»:
  выигрыш не оправдывает вторую индексную инфраструктуру на текущем масштабе.

## 4. Что действительно работает в обучении (доказательная база)

| Принцип | Доказательства | Как применяем |
|---|---|---|
| **Interpolated retrieval practice** (вопросы ВНУТРИ видео, не после) | RCT на 703 студентах (Nature Comm. Psychology, 2025): встроенные вопросы улучшают результаты в университете и колледже; эффект связан с вовлечённостью. Вопросы в конце длинного видео не работают — важна правильная точка. | Вопрос после каждого сегмента, на границе разделов конспекта. Ошибка → существующий flashcard handoff. |
| **Сегментация ~6 минут** (Mayer: segmenting principle; MOOC-исследования Guo et al.) | Вовлечённость падает после ~6 минут непрерывного видео; учащийся должен контролировать темп. | «Видео-сегмент дня» в Course Cockpit; разделы конспекта = главы видеоплеера. |
| **Dual coding** (Paivio; «6 стратегий» Weinstein & Sumeracki) | Слова + образы кодируются двумя каналами → прочнее след памяти. Из шести стратегий hometutor уже покрывает retrieval practice и spacing — dual coding это ровно недостающая мультимодальная. | Слайд/диаграмма рядом с текстом раздела; image-карточки. |
| **Signaling + coherence** (Mayer) | Подсветка ключевого улучшает, декоративное — вредит. «Шедевр» ≠ «нарядно». | Обогащение раздела = термины, схема, пример; никаких стоковых картинок. |
| **ICAP** (Chi & Wylie) | Interactive > Constructive > Active > Passive: диалог сильнее самообъяснения, самообъяснение сильнее конспектирования, конспектирование сильнее просмотра. | Лестница в UI: посмотрел сегмент → ответил на вопрос → объяснил своими словами → обсудил с тьютором. SSR двигает вверх по лестнице. |
| **Self-explanation** (генеративное обучение) | Устойчивый эффект в мета-анализах. | Промпт «объясните своими словами» после сегмента, оценка тьютором (существующий tutor-контур). |

Анти-паттерны (что НЕ делать, тоже по данным): вопросы «в самом конце» длинного видео; автоплей следующего
сегмента (ломает self-pacing); декоративные иллюстрации (coherence principle); дублирование текста на экране
голосом дословно (redundancy principle).

## 5. User Stories (по эпикам, с acceptance criteria)

### Epic M0 — «Видео живёт в конспекте» (MVP, без новых зависимостей)

**US-M0.1.** Как студент, я хочу привязать видео (YouTube-URL или локальный файл) к конспекту,
чтобы конспект знал свой первоисточник.
*AC:* поле `video:` в frontmatter; «Живой конспект» и просмотр конспекта показывают плеер/ссылку;
битый путь — мягкая деградация в текст ссылки.

**US-M0.2.** Как студент, я хочу видеть в разделе конспекта кнопку «смотреть с этого места»,
если у раздела есть таймкод, чтобы не мотать видео вручную.
*AC:* `st.video(start_time=t)` для локальных файлов, `?t=` для YouTube; таймкоды берутся из
`<konspekt>.media.json`, при его отсутствии кнопка не показывается.

### Epic M1 — «Видео как источник» (ASR-конвейер)

**US-M1.1.** Как студент, я кладу видеолекцию в папку лекции и запускаю одну команду,
чтобы получить транскрипт и собрать конспект как обычно.
*AC:* `scripts/transcribe_media.py` принимает mp4/mkv/mp3/wav; кладёт `.txt` + `.segments.json` рядом;
идемпотентен по sha256; работает offline; ffmpeg отсутствует → понятная ошибка с инструкцией;
`gather_lecture_inputs` подхватывает результат без правок.

**US-M1.2.** Как студент, я хочу, чтобы разделы сгенерированного конспекта автоматически получили таймкоды,
чтобы каждая тема была кликабельной точкой видео.
*AC:* post-hoc выравнивание `own_text`↔сегменты; ≥80% разделов конспекта часовой лекции получают таймкод;
регрессионный тест на детерминированность выравнивания.

**US-M1.3.** Как студент, я хочу, чтобы транскрипт попал в общий индекс,
чтобы «Быстрый ответ» находил сказанное в видео и отвечал со ссылкой на минуту.
*AC:* chunk metadata несёт `t_start`; карточка источника показывает «▶ 12:34».

### Epic M2 — «Раздел-шедевр» (обогащение)

**US-M2.1.** Как студент, я хочу видеть слайды презентации внутри соответствующих разделов конспекта,
чтобы текст и образ работали вместе (dual coding).
*AC:* PDF → PNG постранично (pymupdf); привязка слайда к разделу по OCR/подписи VLM;
рендер `st.image` в разделе; выключаемо флагом.

**US-M2.2.** Как студент, я хочу кнопку «Обогатить раздел»,
чтобы скучный текст превратился в объяснение с диаграммой, аналогией и конкретным примером.
*AC:* LLM генерирует: mermaid-схему связей, 1 аналогию, 1 конкретный пример, 3 выделенных термина;
результат — предпросмотр с принять/отклонить (паттерн flashcards preview); coherence-правило в промпте
(«ничего декоративного»); промпт в `app/prompts/`.

**US-M2.3.** Как студент, я хочу, чтобы диаграммы и скриншоты из материалов имели текстовые подписи,
чтобы поиск находил «ту самую схему».
*AC:* VLM-подпись через `get_vision_llm()`; подпись индексируется; фича off по умолчанию
(`VISION_CAPTIONS_ENABLED`).

### Epic M3 — «Интерактивный просмотр» (учебный цикл внутри видео)

**US-M3.1.** Как студент, я хочу, чтобы после каждого сегмента видео мне задавали один вопрос по только что
услышанному, чтобы не смотреть «в пустоту» (interpolated retrieval).
*AC:* вопрос генерируется из транскрипта сегмента (существующий quiz-генератор, новый scope);
ошибка → карточка через существующий handoff; частота — не чаще 1 вопроса / 5–7 минут.

**US-M3.2.** Как студент, я хочу после блока сегментов объяснить тему своими словами и получить разбор,
чтобы подняться от «посмотрел» к «понял» (ICAP: constructive).
*AC:* self-explanation промпт → оценка тьютором → слабые места в learner state.

**US-M3.3.** Как студент, я хочу, чтобы приложение помнило, какие сегменты я посмотрел и где ошибался,
чтобы прогресс по видео был частью общего прогресса.
*AC:* таблица `media_progress` через `user_state*`-слой; сегменты со статусом
unwatched/watched/verified (verified = ответил на вопрос сегмента).

### Epic M4 — «Курс-плеер» (активный курс)

**US-M4.1.** Как студент на активном курсе, я хочу получать «видео-сегмент дня» (≤6 минут) с follow-up
вопросом в ротации Course Cockpit, чтобы видео вписалось в мой ежедневный ритм.
*AC:* новый слот в `cockpit_rotator`; сегмент выбирается по плану курса и статусу `media_progress`.

**US-M4.2.** Как студент, я хочу, чтобы SSR при слабом mastery темы рекомендовал пересмотреть конкретный
сегмент видео (а не «весь материал»), чтобы ремедиация была точечной.
*AC:* новый `hint_kind=video_segment` с evidence «тема X слабая + сегмент 12:34–18:02 не verified»;
детерминированный сигнал, в духе существующего SSR.

## 6. Customer Journey Maps

### CJM-1: «Из видеолекции — в шедевр-конспект» (maintainer-путь, раз в лекцию)

| Стадия | Действие студента | Точка контакта | Эмоция сейчас → цель | Барьер | Метрика успеха |
|---|---|---|---|---|---|
| 1. Сброс | Кладёт `The_Architecture_of_Autonomy.mp4` в `materials/<курс>/<лекция>/` | Проводник / Mission Control подсказка | «опять час смотреть» → «сейчас машина посмотрит за меня» | не знает про папку → first-run подсказка «нашёл видео без транскрипта» | видео обнаружено, предложено действие |
| 2. Транскрибация | Одна команда / одна кнопка | `transcribe_media.py` / Mission Control action | ожидание → «ого, 5 минут на часовую лекцию» | нет ffmpeg → preflight с инструкцией | `.txt`+`.segments.json` созданы; время < 10 мин/час видео |
| 3. Конспект | Обычный smart-konspekt запуск | существующий конвейер | привычно | длинный транскрипт → map-reduce уже решает | конспект с таймкодами ≥80% разделов |
| 4. Обогащение | «Обогатить раздел» на 2–3 ключевых разделах | Живой конспект, preview | «скучный текст» → «мой учебник с картинками и схемами» | перегрузить декором → coherence-правило в промпте | принято ≥1 обогащение; отклонения ≤50% |
| 5. Учёба | Разделы уходят в quiz/cards/план | существующий цикл | «конспект лежит мёртвым» → «конспект учит меня» | — | сегменты → verified; карточки из ошибок |

### CJM-2: «Ежедневная сессия с интерактивным видео» (learner-путь, каждый день)

## 8. Риски и анти-цели (согласовано со стратегическим обзором 2026-07-03)

1. **Не строить вторую индексную инфраструктуру.** Мультимодальный retrieval достигается подписями (текст) в
   существующий BM25/vector — Qwen3-VL-Embedding отложен до реальной потребности.
2. **Не тянуть тяжёлые зависимости в core.** faster-whisper — extra `asr`; ffmpeg — системная с preflight-проверкой;
   VLM — вообще без python-зависимости (endpoint).
3. **Не плодить модули.** Бюджет: фаза 0 — 0 новых модулей (правки в living_konspekt_view + section rendering);
   фаза 1 — 2 (`scripts/transcribe_media.py`, `app/media_alignment.py`); фаза 2 — 2–3. Size-гвард обязателен зелёный.
4. **Не делать «нарядно».** Coherence principle: обогащение = схема/пример/аналогия, не декор.
   Промпт-правило + preview с отклонением.
5. **Хранение медиа — вне git.** Кейфреймы/слайды в `data/media/` (gitignored), в конспекте — относительные пути.
6. **Каждая фаза — за фичефлагом** (`MEDIA_*`), off по умолчанию до стабилизации, в духе `ingest_docling_enabled`.
7. **Приватность.** ASR и VLM локальные; облачные — только явный opt-in, как и весь LLM-контур.

## 9. Что упущено в исходных набросках (добавлено этим планом)

- **Interpolated retrieval как главный механизм интерактивности** — не «видео в конспекте», а «вопросы в видео»:
  именно это по данным RCT даёт прирост обучения, а не сам факт наличия плеера.
- **Сегмент ≤6 минут как единица курса** — видео входит в ежедневный ритм Cockpit, а не как «посмотри часовую лекцию».
- **ICAP-лестница** как дизайн-принцип UI: каждая поверхность должна предлагать шаг вверх (смотрю → отвечаю → объясняю → обсуждаю).
- **Выравнивание раздел↔таймкод без LLM** — детерминированный алгоритм на уже существующем скоринге, тестируемый.
- **`media_progress` как новый сигнал SSR** — мультимодальность усиливает существующий router, а не живёт сбоку.
- **Анти-цели** — явный список того, что делать не нужно, чтобы план не превратился в источник нового расползания.
