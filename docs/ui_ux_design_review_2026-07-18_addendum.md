## 1. Зачем отдельный документ, а не правка ревью

Baseline-аудит и план уже верны и не пересматриваются. Три новых сигнала не
меняют baseline-приоритеты: W1 quiz integrity уже реализован в runtime и
остаётся release-validation gate, а блокирующий онбординг и accessibility
остаются самыми острыми открытыми дефектами продукта. Документ добавляет
конкретику к трём областям, которые в baseline описаны на уровне вердиктов
без детального дизайн-решения: тема/палитра, типографика, вес и состав
главной страницы.

## 2. Общая оценка — уточнение

| Критерий | Baseline | Дельта | Комментарий |
|---|---:|---:|---|
| Cohesion / Brand | 6,2 | **5,8** | У продукта нет «дневного голоса»: неон зала, бумага разборов и безликий фиолетовый Streamlit-каркас между ними не связаны |
| Modernity & delight | 7,0 | 7,0 | Зал тянет вверх (Memory Run — сильная сцена), дневной UI вниз |
| Clarity & hierarchy | 5,7 | **5,4** | Главная: 6+ равновесных одинаковых по весу CTA на одном экране |
| Accessibility | 5,4 | 5,4 | Без изменений; W1 реализован, но release gate ещё нужен; onboarding/accessibility остаются открытыми |
| Performance feel | 6,3 | 6,3 | Без изменений |
| **Итог** | 6,1 | **6,0** | Диагноз сузился: не «что показывать», а «каким весом и каким голосом» |

**Тезис дельты.** У hometutor уже есть два выдающихся визуальных языка:
spatial-неон Memory Run (`--kgx-*`) и тёплая бумага notebook-разборов
(paper/terra/gold + Georgia). Оба любимы пользователями. Провал — в третьем,
безличном языке дневного Streamlit-UI, где единственный строительный блок —
градиентная кнопка-плита одного веса. Пользователи выбирают «Космос» не
потому, что им нужен фиолетовый — это единственный мир с характером,
совпадающим с неоном зала. Решение — не искать шестую палитру, а дать
дневному UI два родных характера, которые у продукта уже есть.

## 3. Мнемополис — подтверждение + два уточнения к W5

Зал остаётся лучшей поверхностью продукта и эталоном «честной проекции
данных»; W5 (`ui_ux_design_review_implementation_plan.md`, editorial
reduction) остаётся верной волной без изменений scope. Два уточнения по месту
работ:

- **Embedded-хром хуже экспортного.** Над сценой в runtime — до 10 кнопок
  парами «обычная / LLM» (Экскурсия, Угрозы, Цель, Голоса, Летопись) плюс
  dev-строка вида `экскурсия:degrade · угрозы:degrade(0)…`. Это подтверждает
  находку W5 о перегрузе первого кадра и уточняет адрес: пара «X / X LLM» —
  один переключатель, не две кнопки; `degrade`-строка переходит в Expert
  layer, а не остаётся в normal chrome.
- **Зелёный CTA «Начать» — образцовая находка, которую стоит унести наружу.**
  Единственный тёплый акцент в фиолетово-неоновом мире безошибочно маркирует
  главное действие. Правило «один цвет = одно главное действие на экран»
  сейчас нарушено во всём дневном UI (см. §6) и должно перейти туда как
  semantic token, а не как визуальный трюк зала.

## 4. Палитра и миры — основная находка

### 4.1 Диагноз (код + скриншоты)

1. **Пять миров — один шаблон с заменой оттенка.**
   [app/ui/theme_presets.py](../app/ui/theme_presets.py) устроен образцово как
   механизм — единственный источник истины, полный набор токенов на мир,
   валидация в `tests/test_theme_presets.py` — этот механизм сохраняется без
   изменений. Проблема в значениях: у всех пяти миров одна и та же формула
   градиентных плит (`head-*`, `button-bg`), встречаются откровенно
   рассинхронизированные миксы — у мира «Лес» токен `head-due` уходит в
   фиолетово-терракотовый градиент внутри «зелёного» мира. Отличаться хуже,
   чем не отличаться: 4 из 5 миров корректно бракуются пользователями.
2. **Кнопка-плита делает любую тему кричащей.** `.stButton > button` в
   [app/ui_theme.css](../app/ui_theme.css) (район строк 286–297) — это всегда
   градиент + `font-weight: 800` + белый текст + тень, вне зависимости от
   важности действия. Герой SSR и третичная кнопка на плитке каталога весят
   одинаково — это одновременно причина ощущения «кричаще» в мирах и
   «простецки» на главной (см. §6).
3. **Витрина выбора темы: raw `**title**` был подтверждённым живым P2-багом,
   не historical/needs verification.** Причина — `**{title}**`, приклеенный
   сразу после блочного `<div>` в едином
   `st.markdown(..., unsafe_allow_html=True)` в
   [app/ui/control_panel.py](../app/ui/control_panel.py): CommonMark HTML-block
   поглощал остаток строки как raw-текст, и markdown-bold не парсился. Баг
   закрыт 2026-07-18: заголовок рендерится как HTML `<strong>...</strong>`,
   добавлен regression-тест. Оставшийся P2-polish по этой витрине — карточки
   миров почти не показывают собственный мир, потому что различие считывается
   в основном по узкой цветной полоске и одному символу, а не по полноценной
   мини-витрине палитры.

### 4.2 Вердикты

- **«Космос» → флагман, с рефайном.** Фиолетовый становится акцентом, а не
  тотальной заливкой: нейтральный светлый грунт с холодным оттенком-bias,
  фиолетовый — на primary CTA и активных состояниях; неон-глоу остаётся
  привилегией зала и в дневной UI не переносится.
- **Новая тема «Хроника» — вторая визитная карточка**, ровно то, что просят
  пользователи: локон notebook-разборов как есть — `paper #f7f3ea /
  ink #17211b / terra #b95631 / gold #c98a3d / forest #243b2c` + Georgia для
  заголовков. Палитра уже зафиксирована и провалидирована 20+ разборами —
  ноль нового изобретения, только перенос механизмом THEME_TOKENS.
- **Лес / Океан / Закат / Ягода → понизить до акцент-вариантов** одного из
  двух грунтов («Космос-нейтраль» или «Хроника-бумага»), не отдельных полных
  миров. Формула «2 характера × N акцентных цветов» дешевле в поддержке пяти
  посредственных миров и попадает в запрос пользователей точнее.
- **НЕ делать:** шестую полную палитру с нуля; тёмные варианты миров до
  прохождения portal gate из W3 (уже зафиксировано планом, здесь не
  меняется).

### 4.3 Направление токенов (иллюстративно, не финальный diff)

```css
/* «Хроника» — консервативный характер, перенос notebook-локона */

"ink": "#17211b", "muted": "#5c6a61",
"accent": "#b95631",
"button-bg": "#b95631",          /* плоский цвет, без градиента */
"head-text": "#fffdf7",

/* «Космос» refined — фиолетовый как акцент, не заливка */

```

## 5. Типографика — голос разборов как визитная карточка

Сейчас в приложении только системные стеки — `--font-sans` для текста и
`--font-mono` для кода (`app/ui_theme.css`); display/serif-роли не существует.
При этом шрифты, которые любят пользователи, — это Georgia + системный sans,
уже используемые в notebook-разборах: не экзотика, доступны без вендоринга,
совместимы с local-first.

**Направление — двухголосая типографическая система:**

```css
:root {
  --font-display: Georgia, "Times New Roman", serif; /* заголовки и лиды */
  /* --font-sans без изменений — body, controls, данные */
}
.page-kicker { text-transform: uppercase; letter-spacing: 0.16em;
               font: 700 0.72rem var(--font-sans); color: var(--accent); }
.page-title  { font: 700 1.65rem/1.15 var(--font-display);
               letter-spacing: -0.015em; text-wrap: balance; }
.page-lede   { font-size: 1.05rem; color: var(--muted); max-width: 64ch; }
```

Это анатомия шапки notebook-разбора (kicker → serif-заголовок → lede),
оформленная как переиспользуемый компонент `PageHeader` — сам компонент уже
называн в §14 baseline-ревью; этот раздел фиксирует его конкретный визуальный
контракт. Границы применения: serif — только заголовки/лиды и читальные
поверхности (полный notebook-режим — кандидат для читалки Living Konspekt в
теме «Хроника»); body, controls и данные остаются sans; числа — tabular;
Мнемополис типографику не меняет — у зала свой голос.

## 6. Главная страница — состав верный, вес неверный

Код уже проектирует правильную структуру: герой + не более двух resume-карт,
остальное под «Ещё режимы» (`app/ui/mission_control.py`, функция
`render_mission_control`). Живой скриншот показывает, почему это не
считывается:

1. **Шесть и более равновесных CTA на одном экране** — «Повторить», «Открыть
   граф знаний», «Открыть Темы», плюс сайдбар («Мой прогресс», «Продолжить
   обучение», «Восстановить курс») одного визуального веса. Проблема не в
   количестве, а в одинаковости.
2. **«Восстановить курс: physics» дублируется** — в сайдбаре и в плиточной
   сетке одновременно. Один continuity-CTA, одно место.
3. **First-session артефакт «Первая сессия: living-konspekt»** с тремя
   заглушками «Черновик ответа» показывается пользователю с XP 6862 —
   должен гейтиться состоянием cold/returning, которое код уже различает по
   другим признакам.
4. **Псевдо-кнопка-подпись** «Выберите стартовый вопрос ниже» стилизована как
   CTA, будучи меткой — метка не должна носить одежду кнопки.
5. **Серые чипы `Q&A · Tutor · Quiz · Card · Review · Graduation`** — это
   внутренние имена этапов без пользовательского перевода, выглядят как
   отладочная строка в normal layer.
6. **«0.0% MASTERY»** крупным числом в первый день демотивирует правдивым, но
   плохо поданным нулём: до первого квиза корректнее показывать состояние
   «старт сегодня», не процент.

**Направление — трёхуровневая система весов кнопок** (входит в W3 как
control tiers и чинит проблему во всём приложении разом, не только на
главной):

```css
.btn-primary   { background: var(--accent); color: #fff; }        /* одна на экран */
.btn-secondary { background: var(--accent-soft); color: var(--accent-dark);
                 border: 1px solid var(--accent); box-shadow: none; }
.btn-tertiary  { background: none; border: none; color: var(--accent);
                 text-decoration: underline; min-height: auto; }
```

На главной primary остаётся только у SSR-героя; «Открыть граф/Темы» —
secondary; сайдбар-навигация — tertiary. Дополняет уже зафиксированный в
baseline вердикт: карточка Mission Control должна стать одной кликабельной
поверхностью, а не карточкой с отдельной кнопкой под ней.

## 7. Другие разделы — коротко, без нового приоритета

- **Library / Mega-bundle catalog.** Сетка 3 колонки и адреса «курс · #N ·
  N док.» уже на месте (соответствует W8 и `mega_bundle_catalog_plan.md` в
  `hometutor-studio`). Полировка: пустое поле поиска без плейсхолдера;
  «Спросить» обрезается на третьей карточке при узкой колонке; пара
  «Активным/Спросить» — снова две равновесных кнопки, нужна связка
  primary/secondary из §6; переключатель «Каталог/Пересадки/Маршрут» стоит
  перевести на сегмент-контрол вместо текущего radio-паттерна.
- **Выбор темы.** `**`-bug из скриншотного сигнала был подтверждён живым и
  закрыт 2026-07-18 (HTML-заголовок + regression-тест, см. §4.1). Оставшаяся
  проблема: карточки миров не показывают сам мир; различие видно только по
  узкой цветной полоске и одному символу. Карточка темы должна быть
  мини-витриной собственной палитры.
- **Quiz / Onboarding.** W1 quiz leak реализован в runtime; до release gate его
  надо держать как validation item, а не как открытый дизайн-дефект. Блокирующий
  onboarding без якорей и accessibility baseline остаются первичными открытыми
  дефектами по отношению к палитровым и типографическим предложениям.

## 9. Метрика и шестой gate

**Метрика восторга.** Сегодня неизмерима. После W3b и телеметрии удержания
темы — измерима двумя числами: доля новичков, оставшихся на выбранной теме
≥14 дней, и доля выбравших не-дефолтную тему (сигнал, что выбор осознанный,
не случайный).

**Шестой gate к пяти из baseline-ревью §15:** «тест двух голосов» — каждая
новая поверхность обязана явно выбрать характер (spatial-неон зала / бумага
«Хроники» / нейтральный рабочий грунт) и не смешивать их в одном блоке.
Кнопка уровня `primary` — не чаще одной на экран; всё, что не главное
действие, не получает градиент.

  W3, как и прежде.
- Редактирование волн, write-set'ов и Definition of Done в
  [ui_ux_design_review_implementation_plan.md](ui_ux_design_review_implementation_plan.md) —
