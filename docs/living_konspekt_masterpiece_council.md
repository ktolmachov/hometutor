# Живой конспект: критерии шедевра и открытый стол пользователей

Статус: продуктовый handoff-документ (самодостаточный). Подготовлен: 2026-07-06, после
верификации W4. Ревизия: 2026-07-06 (вторая) — по встречному аудиту: W5 к этому моменту
уже реализован (`app/konspekt_artifact.py` + UI), оценки Ставки 3 и «единой валюты»
приведены к коду, события метрик помечены как вводимые — см. журнал в конце.
Жанр: не инженерный план, а проверка «это уже отлично — но шедевр ли?» глазами
пользователей и по критериям категории. Технический план волн остаётся источником
истины по контрактам; этот документ — источник истины по **смыслу и приоритету ставок**.

---

## 1. Категория и планка

Категория: **локальный учебный компаньон** — конспекты + видео лекций + повторение +
курс. Продукты, которые студенты реально любят, и за что именно:

| Продукт | За что его любят (одним словом) |
|---|---|
| Anki | **Доверие** — алгоритм честен, труд не пропадает |
| Obsidian | **Владение** — мои файлы, моя структура, никакого lock-in |
| YouTube ×2 | **Ноль трения** — нажал и смотришь, таймкоды в комментах |
| Duolingo | **Привычка** — причина вернуться завтра |
| Google Maps | **Одна метафора** — гигантская сложность за словом «маршрут» |

Шедевр в этой категории — не «больше фич», а продукт, который берёт по одному
сильнейшему свойству из каждой строки и не теряет ни одного.

## 2. Критерии шедевра (чек-лист по текущему состоянию)

| # | Критерий | Почему решает | Сейчас | Разрыв |
|---|---|---|---|---|
| 1 | **Одна метафора**: объясняется однокурснику одной фразой | Продукт, который нельзя пересказать, не рекомендуют | ❌ | «RAG-приложение с 8 пунктами меню» — фразы нет |
| 2 | **Первый «вау» ≤ 3 минуты** от запуска | Студент не даёт второго шанса | 🟡 | first_run есть, но «вау» (видео-момент из СВОЕЙ лекции) спрятан в глубине |
| 3 | **Владение трудом**: локально, markdown, без lock-in | Категория живёт на недоверии к облакам | ✅ | vault, local-first, W4 переносимость — сильнейшая сторона |
| 4 | **Каждое утверждение проверяемо кликом** (строка конспекта / секунда видео) | Anki-уровень доверия, но для ответов | 🟡 | Slim-слой есть: source-card Быстрого ответа показывает trusted video citation при уверенном resolver + актуальном sidecar. `anchor-lis-v3.1` (2026-07-07) поднял покрытие confident-таймкодов: на урок_2 медианная ошибка контрольных точек 233 c → 86 c, катастрофические промахи (до 39 мин) устранены; L1-синонимия закрывает типовой разрыв «практическое задание» ↔ «домашка/упражнение». | 5 | **Экономия времени измерима**: «2 часа лекции → мои 18 минут» | Главная валюта студента | 🟡 | Таймкоды по разделам есть; плейлист-метрика стала честной (`anchor-lis-v3.1`, объединение интервалов проходов вместо двойного счёта: было «плейлист 3:13 из 1:57», стало «1:37 из 1:57»). «Смотреть только собранное» как режима — по-прежнему W5.5/W6 slim |
| 6 | **Причина вернуться завтра** без манипуляций | Привычка = выживание фичи | 🟡 | «Память конспекта» (due по source:-тегам) — честный механизм, но виден только внутри фичи |
| 7 | **Выручает в ночь перед экзаменом**: выжимка, печать, офлайн | День паники — день максимальной благодарности | 🟡 | Печать реализована (W5, `open_print_view` из фичи); режим «выжимка к четвергу» (отбор фрагментов по списку вопросов) не специфицирован ни в одной волне — парковка, см. §6 |
| 8 | **Прощает хаос** во входных материалах | Реальные лекции — это бардак | 🟡 | Non-portable snapshot и honest degradation — хороший фундамент |
| 9 | **Труд не теряется никогда** | Одна потеря корзины = удаление приложения | ✅ | W4: автоперсист, row_key, переносимость. W5 реализован (2026-07-06): артефакт живёт — manifest, save-as-upsert, «Мои конспекты», пересборка в корзину |
| 10 | **Результатом можно гордиться и делиться** | Рост без маркетинга | ❌ | Сохранённый конспект — файл в vault, не «вещь», которую показывают однокурснику |

**Счёт (после реализации W5): 2 ✅ · 6 🟡 · 2 ❌ — «уже отлично, ещё не шедевр».**
Диагноз совпадает с ощущением владельца. Ключевое: оба ❌ (одна метафора, «чем
гордиться и делиться») — это не отсутствие технологий, а отсутствие **одной сцены
и одного языка** поверх уже построенного.

## 5. Сильный ход: «Хребет фрагментов» — три ставки

### Нестандартность хода

Привычный путь — добавлять фичи. Нестандартный — признать, что **большая часть уже
построена, но говорит на разных языках**. Точная картина по коду — это **две трубы,
а не одна валюта**:

- **Труба медиа-якорей:** `section_id` (`media_alignment.compute_section_id`) —
  контентный якорь раздела, который уже связывает конспект ↔ таймкоды видео
  (media-sidecar) ↔ строки корзины/манифест артефакта ↔ карточки (source:-теги).
- **Труба retrieval-резолва:** `top_sections_for` (`section_index.py`) — лексический
  token-overlap «текст запроса/чанка → раздел». Она НЕ использует `section_id`,
  не возвращает score наружу и не имеет порога уверенности.

«Хребет фрагментов» — это **продуктовая цель склейки двух труб**, а не текущее
состояние кода. Ход — сделать хребет видимым и назвать его словом, понятным
первокурснице: **фрагмент** — смысловой кусок лекции, у которого всегда есть текст,
минута видео, карточки и соседи.

Проверка «что уже есть / что строить»:

- «Нарезка видео» — уже работает без ffmpeg и перекодирования: `st.video(start_time=)`
  для локальных файлов, YouTube-embed с таймкодом (`living_konspekt_media.py`).
  Нарезка — виртуальная, по таймкодам sidecar'а. Строить: **плейлист-UI** поверх.
- «Ответ с видео-цитатой» — slim-слой реализован в W7: guarded resolver даёт
  source→section, а отдельный слой resolver + sidecar показывает кликабельный timestamp
  только при актуальном sidecar и достаточном confidence. Покрытие честное, но частичное:
  источник из документа без media-sidecar (нет ASR-выравнивания) даёт текстовую цитату
  без таймкода — деградация, не баг.
- Правило доверия (stale/low-confidence → таймкод не показываем) — уже есть и
  распространяется на все три ставки без изменений.

### Ставка 1 — «Мои 18 минут» (плейлист фрагментов)

В Живом конспекте появляется кнопка **«▶ Смотреть собранное»**: корзина превращается
в плейлист видео-фрагментов — по таймкодам разделов, в порядке глав пользователя.
Двухчасовая лекция → 18 минут именно того, что студент отобрал. Это ответ Соне
(crammer) и главный измеримый «вау»: экран показывает «Собрано: 18 мин из 2 ч 07 мин».
Фундамент готов: manifest уже несёт указатели на sidecar'ы (`collect_sidecar_pointers`,
реализовано в W5) — но сам плейлист-UI и подсчёт длительности **не входят в W5**,
это работа W5.5/W6 (§6). «Мои конспекты» станут «моими плейлистами» именно там.

### Ставка 2 — Синхро-чтение

Reader перестаёт быть текстовой простынёй: у каждого раздела — кнопка «смотреть этот
кусок», плеер открывается на нужной секунде **не покидая читалку**; TOC + прогресс
«прочитано 7/20» (уже в W5/W6 плана). Читаешь — можешь смотреть; смотришь — видишь
текст. Это сердца Леры и Даши: язык видео — родной язык поколения, текст подтягивается
за ним, а не наоборот.

### Ставка 3 — «Спроси лектора» (хейтер-конвертер)

В «Быстром ответе» источник ответа получает не только «файл:строки», но и
**кликабельную видео-цитату** при достаточном доверии: «🎬 Смотреть с 41:20» — ссылка
открывает видео на месте. Цепочка реализованного slim-слоя: sources ответа → guarded
resolver в раздел → `section_id`/line fallback → актуальный sidecar-таймкод → ссылка.
Ниже порога, при дублях/ручном выборе без подтверждения или без sidecar — честно без
таймкода. Ценность не отменяется: этого не может ChatGPT — он не знает, где ВАШ
преподаватель сказал это, и не откроет видео на этой секунде. Плюс на карточке
источника — «➕ В конспект» (§4.5): вопрос → ответ → фрагмент → корзина. Петля замкнулась.

### Одна фраза (тест критерия №1)

> «Это навигатор по твоим лекциям: задаёшь вопрос — он открывает видео на нужной
> секунде и абзац конспекта, а из лучших фрагментов ты собираешь свой маршрут к экзамену.»

Слова «граф», «RAG», «воркбенч» в фразе не нужны — и это признак, что метафора найдена.
Честная оговорка: фраза-тест (§7.1) может пройти не раньше Ставки 1 — «собираешь маршрут
из фрагментов» требует плейлиста, а не только переименования словаря UI в W5.

**Парковка (решения без исполнителя — явно):** режим «выжимка к четвергу» (Соня: отбор
фрагментов по списку вопросов → мои 18 минут под конкретный экзамен) — не специфицирован
ни в одной волне; кандидат на спецификацию после Ставки 1, из которой он собирается
почти целиком. До спецификации — не обещание.

Конституция (по Нине и анти-целям плана): локальность по умолчанию, LLM — только
opt-in, ядро всех трёх ставок детерминированно (таймкоды и резолв — без LLM-вызовов).

## 7. Как поймём, что стало шедевром

1. **Тест одной фразы**: 5 студентов пересказывают приложение после 3 минут — совпадает
   ли пересказ с фразой из §5.
2. **Время до первого фрагмента** (событие `living_konspekt_section_added`) < 3 минут
   от первого запуска.
3. **Возврат**: доля пользователей с повторным открытием сохранённого конспекта.
   События `artifact_reopened`/`artifact_rebuilt` введены в W5 slim-срезе рядом с уже
   живым `living_konspekt_saved` (`ui_events.track_event`).
4. **Хейтер-конверсия**: доля кликов по видео-цитате в ответах. Slim-слой Ставки 3
   показывает trusted citation (`ask_lecturer_video_citation_shown`) и в local no-auth
   режиме пишет фактическое открытие через redirect (`ask_lecturer_video_citation_clicked`).
5. **Экономия времени на экране**: суммарное «собрано N мин из M ч» — цифра, которой
   студент хвастается сам, без нашего маркетинга.

начала/конца смысловых блоков — ключ; узлы смыслов видео в графе»). Затрагивает критерии

- **Смысловые блоки** (`app/media_alignment.py::build_semantic_blocks`): транскрипт режется
  TextTiling-подобной сегментацией по провалам лексической связности. У блока настоящие
  границы темы (`t_start`/`t_end`) и ключевые слова (топ TF-IDF). Экспорт в sidecar —
  `semantic_blocks` (опциональное поле схемы v1, старые sidecar валидны).
- **per-pass хронология** сняла структурное ограничение прошлой ревизии: конспект — это
  несколько «проходов» по одной лекции (H2-группы: слайды → ключевые темы → примеры),
  монотонность таймкодов держится внутри прохода, между проходами независима. Глобальная
  единственная возрастающая цепочка (v1/v2) прижимала recap-проходы к хвосту лекции — v3
  этого больше не делает.
- **Точность** (урок_2, 10 ручных контрольных точек против транскрипта): медианная ошибка
  233 c (v1) → 86 c (v3); катастрофические промахи (Слайд 46: −39 мин; MCP: −21 мин)
  устранены (−13 c и +81 c соответственно). 5/9 точек ≤90 c против 2/10 у v1.
- **Честная плейлист-метрика**: `_coverage_metrics` считает объединение интервалов
  confident-фрагментов. Раньше проходы, указывающие на одни минуты, давали «плейлист
  3:13 из 1:57 медиа»; теперь «1:37 из 1:57». `t_end` больше не растягивается до конца
  медиа у последнего раздела прохода.
- **Узлы смыслов видео в графе** (`living_konspekt_next_steps.video_semantic_moments` +
  граф-линза): блоки, пересекающиеся с собранными фрагментами, показываются как
  «🎬 Смыслы видео рядом» с таймкодом `t_start–t_end` и ключевыми словами. Детерминированно,
  без LLM, деградирует молча при отсутствии sidecar; с `anchor-lis-v3.1` уважает тот же
  trust-gate, что media panel/export: stale sidecar и low-confidence разделы не дают
  video moments.
- **L1-синонимия без LLM**: статический локальный словарь учебных паттернов помогает якорить
  разделы вроде «Практическое задание», когда в речи звучит «домашка/попробуйте сами/упражнение».
  Это не общий semantic search: расширяются только scoring-токены и только словами, реально
  найденными в ASR.
- Тесты: `tests/test_media_alignment.py` переписан под иерархические (многопроходные)
  фикстуры — сегментация, per-pass немонотонность, инварианты `t_end`; +schema-тесты
  `semantic_blocks`, L1-синонимия и trust-gate граф-линзы покрыты targeted media-тестами.
- **Осталось честно открытым**: широкий semantic matching за пределами L1-словаря

  с явным provenance; ядро остаётся детерминированным.
