Актуализировано: 2026-07-03.

## Для кого это

`hometutor` полезен, если у вас есть лекции, конспекты, статьи, Obsidian vault или учебные PDF, и вы хотите не просто “поговорить с документом”, а пройти цикл обучения.

```text
data/ с материалами
  -> индекс
  -> ответ с источниками
  -> tutor-разбор
  -> quiz
  -> flashcards + SM-2
  -> progress + Smart Study Router
  -> Knowledge Graph -> точные разделы -> Живой конспект
```

Проект local-first: материалы, индекс и учебное состояние находятся на вашей машине. По умолчанию `config.env` направляет LLM и embeddings на loopback endpoints (`127.0.0.1`). Если вы явно переопределяете `EMBED_API_BASE`, `LLM_API_BASE` или fallback-настройки на облачный provider, содержимое запросов может уходить внешнему провайдеру (см. [quickstart.md](quickstart.md)).

## Первый запуск

При первом входе `hometutor` проверяет три вещи до вопроса: доступен ли API, есть ли материалы в индексе и отвечает ли локальная модель. Если API недоступен, Mission Control показывает действие для запуска сервера. Если модель выключена, вы увидите подсказку запустить LM Studio или совместимый OpenAI-compatible сервер и загрузить модель из настроек.

Если индекс пуст, главная показывает три входа:

- **Загрузить файлы** — выберите `.md`, `.txt`, `.pdf`, `.docx` или `.html`; UI сохранит их в `data/uploads/` и запустит переиндексацию.
- **Попробовать демо-материалы** — учебные файлы из `demo_data/` копируются в `data/demo/`, после индексации появятся готовые стартовые вопросы.
- **У меня уже есть папка** — положите материалы в показанный путь `data/` и нажмите «Переиндексировать».

Демо-режим помечается баннером «демо-песочница». Его можно удалить одной кнопкой: если после удаления не осталось материалов, активный индекс очищается до пустого состояния, чтобы демо-документы больше не участвовали в ответах.

## Главная: Mission Control

Первый экран — не landing page, а рабочая панель. Она показывает:

- Smart Study Router: один рекомендуемый следующий шаг и объяснение “почему сейчас”;
- resume-карточки для тьютора, due-карточек и активного курса;
- быстрые плитки режимов;
- доступ к Knowledge Graph, истории, метрикам и explain-file.

Главная нужна для возврата: пользователь не вспоминает, где остановился, а продолжает с локального состояния.

## Панель управления и уровни интерфейса

Интерфейс может показывать разный объём возможностей. В сайдбаре и на Mission Control есть кнопка
«Настроить интерфейс»: она открывает панель с пресетами.

| Уровень | Что показывает |
|---|---|
| 1 Начальный | Mission Control, быстрый ответ, поиск материалов, объяснение файла |
| 2 Основной | учебные режимы: тьютор, quiz, flashcards, прогресс, темы |
| 3 Продвинутый | курс, адаптивный план, Knowledge Graph, живой конспект, история |
| 4 Профи | метрики, чистый вид, аналитика, backup/QR-перенос, экспертные Q&A-фильтры |
| 5 Эксперт | debug/trace и диагностические панели |
| Всё включено | полный интерфейс без скрытия разделов |

Существующие пользователи автоматически получают режим «Всё включено», чтобы привычный интерфейс не менялся.
Новому пользователю можно выбрать режим в онбординге: «Начинаю с нуля», «Учусь регулярно» или «Показать всё».

Скрытый раздел не становится недоступным: deep-link, переходы с карточек и «Ещё разделы» всё равно открывают нужную
страницу. В панели есть «Тонкая настройка» — отдельные тумблеры поверх выбранного пресета. Эти настройки хранятся
локально в профиле (`app_kv`) и входят в backup/restore.

## Быстрый ответ

Режим для вопроса “что говорится в моих материалах?”.

Вы получаете:

- ответ по индексу;
- список источников;
- confidence/trace сигналы;
- мост в tutor-режим;
- debug-информацию при включённых диагностических панелях.

HTTP-контракт: `POST /ask`.

## Чат с тьютором

Tutor работает через тот же endpoint `/ask`, но с `query_mode="tutor"` и learner context. Он может:

- объяснить тему проще;
- дать пример;
- встроить micro-quiz;
- продолжить multi-turn сессию по `session_id`;
- учитывать learner goal snapshot.

Важный принцип: переход из ответа в обучение не должен терять тему, источники и цель.

## Quiz

Quiz поддерживает генерацию и оценку вопросов по теме или документу.

Ключевые endpoints:

- `POST /quiz/generate`
- `POST /quiz/evaluate`

Оценка возвращает diagnostic feedback: `recognized`, `recalled`, `misconception`, `cannot_apply`. Эти сигналы участвуют в учебном контуре и могут влиять на следующий шаг.

## Flashcards и SM-2

Flashcards — это human-in-the-loop генерация:

1. выберите источник: документ, upload или active course;
2. сгенерируйте preview;
3. отредактируйте карточки;
4. сохраните колоду;
5. повторяйте по SM-2.

Поддерживается:

- due-очередь;
- оценки `Again / Hard / Good / Easy`;
- recovery tail для большой очереди;
- undo recovery для ещё не повторённых карточек;
- экспорт Anki `.apkg`;
- импорт карточек из quiz.

Ключевые endpoints: `/flashcards/*`.

## Knowledge Graph: точные разделы лекций

Граф знаний — не только карта концептов. У каждого концепта в панели документов показываются **точные разделы** конспекта, где он разобран (до трёх мест: тема, антипаттерны, термины):

- **«📍 «заголовок»»** открывает Obsidian сразу на нужном заголовке конспекта (нужен созданный конспект — кнопка «Подготовить для Obsidian» во вкладке `Темы`);
- **«🖥 VS Code: раздел»** открывает тот же файл на нужной строке;
- при повторяющихся заголовках в документе UI подсказывает, что VS Code-ссылка точнее (Obsidian-якорь открывает первый одноимённый заголовок);
- для `.txt` без конспекта вместо мёртвой кнопки — подсказка подготовить конспект.

В панели «⚡ Действия с концептом»:

- **«➕ раздел «заголовок»»** (до трёх на документ) кладёт раздел в корзину «Живого конспекта»;
- **«➕ Собрать всё по концепту»** добавляет лучший раздел каждого связанного документа одним кликом;
- **«📚 Живой конспект (N)»** показывает счётчик корзины и ведёт в режим сборки.

Такие же ссылки появляются под карточкой во Flashcards: «Открыть раздел» (Obsidian/VS Code) и «➕ В рабочий конспект» — карточка ведёт в точное место лекции, из которого она родилась.

## Живой конспект

Режим `Живой конспект` собирает выбранные разделы разных лекций в рабочий конспект — не теряя контекста и «духа» исходных лекций.

**Корзина разделов** наполняется из графа знаний и из Flashcards (см. выше). Дедупликация — по файлу и строке; корзина живёт в session_state (переживает rerun, но не закрытие вкладки) и восстанавливается через ручное сохранение research-сессии в сайдбаре («Исследования»).

**📚 Собрать рабочий конспект** — два способа:

- *Дословная сшивка (без LLM)*: разделы склеиваются как есть; в шапку добавляется главная мысль каждой исходной лекции, в конец — блок `## Источники` со ссылками `файл:строки`;
- *LLM-синтез из разделов*: модель получает именно выбранные тексты разделов (не повторный поиск по индексу).

Результат сохраняется в `data/living-konspekt/*.md` и попадает в поиск и граф после обновления индекса (UI честно сообщает об этом).

**🌐 Проверить актуальность · источники** — local-first: приложение в сеть не ходит, вы получаете готовые ссылки:

- **«🔗 Ссылки из лекции»** — источники, которые лектор сам приложил к материалу (из текстов разделов и раздела «Дополнительные материалы» конспекта);
- кнопки поисковиков (Google, Scholar, arXiv, Perplexity, YouTube) с коротким запросом из концептов и ключевых слов заголовков.

**🧠 Промпт для глубокого изучения** — копируемый промпт для облачной модели (ChatGPT/Claude/Gemini): главная мысль лекции, дословные цитаты выбранных разделов с источниками, prerequisites/связанные концепты, а также риски/антипаттерны и контрольные вопросы из исходного конспекта. Формулировка требует верности рабочему конспекту и проверки актуальности с источниками.

## Темы и курс

Раздел `Темы` показывает каталог материалов, synthesis и learning plan. Если материалы организованы папками, активируйте папку как course scope и работайте с ней как с курсом.

Course Cockpit показывает daily briefing, pace mode, активность курса и прогресс до graduation.

## Прогресс обучения

Раздел `Прогресс обучения` отвечает на вопрос “что делать сегодня?”.

Там собраны:

- adaptive daily plan;
- mastery;
- analytics;
- weekly narrative;
- due/review состояние;
- Knowledge Graph и weak spots.

Локальное состояние хранится в `data/user_state.db`.

## Smart Study Router

Smart Study Router выбирает следующий шаг из локальных сигналов:

- due flashcards;
- concept-level SM-2;
- свежий ответ с источниками;
- ошибка в quiz;
- weak concept;
- tutor resume;
- active course/adaptive plan.

Результат: `hint_kind`, primary action, secondary actions и объяснение. Feedback `accept/reject/defer` сохраняется локально через `/ssr/recommendation-feedback`.

## Приватность и данные

| Данные | Где |
|---|---|
| исходные материалы | `data/` |
| собранные рабочие конспекты («Живой конспект») | `data/living-konspekt/` |
| прогресс, flashcards, mastery | `data/user_state.db` |
| индекс | `chroma_db/` |
| логи и SSR-профили | `logs/` |
| настройки по умолчанию | `config.env` |
| локальные секреты | `.env` |

По умолчанию `config.env` использует локальные LLM/embedding endpoints. Чтобы ни один фрагмент не покидал вашу машину, держите `EMBED_API_BASE`, `EMBED_MODEL`, `LLM_API_BASE` и `LLM_MODEL` направленными на локальные серверы. При использовании облачного LLM/embedding provider содержимое запросов уходит внешнему провайдеру.
